5.2 Reward Function and Objective Metrics

We reproduce the self-refined reward-design framework on CPU by replacing the original continuous robotic simulator with discrete 2D Gridworld tasks. The observable states are the agent position p=(x,y), goal position, Manhattan distance, shortest-path distance, nearest-lava distance, wall collision indicator, lava collision indicator, timeout flag, and terminal success flag.

The success condition used in the adapted experiment is:

φ ≡ ◇[0,T](p = pgoal) ∧ □[0,T](p ∉ Lava ∧ p ∉ Wall)

This means that the agent must reach the target before the episode terminates while avoiding lava and invalid wall transitions. The objective metrics are: success rate SR, average reward, average number of steps, wall hits, lava hits, shortest-path optimality gap ARPD, and CPU training/evaluation time.

For comparison, we employ a manually designed reward function:

R = 120rgoal - 140rlava - 12rwall - 1 + 6Δdsp - λlava

where Δdsp is progress measured by shortest-path distance. This is the CPU analogue of using task-specific objective metrics as feedback for reward refinement.
